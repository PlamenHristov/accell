05 — Principles

The rules

01

Zinc is the system

The UI lives in the zinc scale (#09090B → #FAFAFA). No blues, purples, or cyans. Background, text, borders — all zinc. The monotone canvas makes the single accent work.

02

One accent, deployed surgically

Orange #F97316 appears only where attention is earned: sensor data, the logo node, active states. If everything glows, nothing does. Use it like punctuation.

03

Components, not pages

Think in cards, rows, and cells. 1px borders. 6-8px radii. Hover states that barely whisper. The system is composable, not decorative.

04

Data is the interface

Joint angles, sensor feeds, latency metrics, system statuses — render in Geist Mono at small sizes. The data isn't decoration. It's the product.

05

Earn every pixel

No gradients. No glows. No shadows heavier than 4px blur. No radius over 12px. Every visual element must justify its existence.

Do

  • Use zinc scale for all neutral surfaces
  • Keep borders at 1px, always
  • Use Geist Mono for data and technical info
  • Reserve orange for sensor data and active states
  • Use DM Sans at weight 300–500 only
  • Keep type sizes 11–15px for most UI
  • Build with cards and rows, not hero sections

Don't

  • Use gradients, glows, or neon effects
  • Add colors beyond zinc + orange + semantic
  • Use type larger than 48px outside of hero
  • Apply border-radius larger than 12px
  • Use serif or display fonts anywhere
  • Add decorative elements that don't serve data
  • Use box-shadow heavier than 0 4px 12px
